Abstract

The Agent2Agent Protocol (A2A) Ontology provides a formal vocabulary for describing interactions between autonomous, opaque agents built on different frameworks. It defines core concepts for tasks, messages, artifacts, and agent capabilities that enable structured communication between agent systems while maintaining their opacity.

Overview

This ontology models the key components of the Agent2Agent Protocol ecosystem:

Agent Card
A description of an agent's capabilities, authentication requirements, and metadata
Task
A stateful entity for collaboration between clients and remote agents
Message
Content exchanged between clients and remote agents
Artifact
Results generated by agents as outcomes of tasks
Part
Individual pieces of content in messages or artifacts

Integration Patterns

The A2A ontology supports agent interoperability through structured task-based interactions. Here are key integration patterns:

Agent Discovery

PREFIX a2a: <http://purl.org/stuff/a2a/>

SELECT ?agent ?name ?description WHERE {
  ?agent a a2a:AgentCard ;
         a2a:name ?name ;
         a2a:description ?description .
}

Skill Discovery

PREFIX a2a: <http://purl.org/stuff/a2a/>

SELECT ?agent ?skill ?desc WHERE {
  ?agent a a2a:AgentCard ;
         a2a:hasSkill ?skill .
  ?skill a2a:description ?desc .
  FILTER(CONTAINS(LCASE(?desc), "customer support"))
}

Task Management Implementation

Tracking and managing task states:

from rdflib import Graph, Namespace, Literal, URIRef
from rdflib.namespace import RDF, XSD
import uuid
from datetime import datetime

A2A = Namespace("http://purl.org/stuff/a2a/")
g = Graph()
g.parse("a2a-store.ttl")

# Create a new task
task_id = str(uuid.uuid4())
task_uri = URIRef(f"http://example.org/task/{task_id}")

g.add((task_uri, RDF.type, A2A.Task))
g.add((task_uri, A2A.id, Literal(task_id)))

# Set initial status
status_uri = URIRef(f"http://example.org/task/{task_id}/status")
g.add((status_uri, RDF.type, A2A.TaskStatus))
g.add((status_uri, A2A.state, A2A.SubmittedState))
g.add((status_uri, A2A.timestamp, Literal(datetime.now().isoformat(), datatype=XSD.dateTime)))
g.add((task_uri, A2A.hasStatus, status_uri))

# Add a message
message_uri = URIRef(f"http://example.org/task/{task_id}/message")
g.add((message_uri, RDF.type, A2A.Message))
g.add((message_uri, A2A.role, Literal("user")))

# Add a text part to the message
part_uri = URIRef(f"http://example.org/task/{task_id}/message/part/1")
g.add((part_uri, RDF.type, A2A.TextPart))
g.add((part_uri, A2A.text, Literal("Tell me about the weather")))
g.add((message_uri, A2A.hasPart, part_uri))

# Connect message to task
g.add((task_uri, A2A.hasMessage, message_uri))

# Save the graph
g.serialize("a2a-store.ttl", format="turtle")

Best Practices

  • Use SPARQL to query for agents with specific capabilities
  • Track task states and transitions in the graph
  • Store messages and artifacts for each task
  • Implement push notification configuration for asynchronous updates
  • Validate data using SHACL constraints
